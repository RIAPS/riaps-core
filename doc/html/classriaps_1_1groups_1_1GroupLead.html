void GroupLead::Update ( )

Maintain the state when no incoming message.

If the node is FOLLOWER and the wait time is not expired, then do nothing

The node is a FOLLOWER but the wait time is expired without hearing from the leader FOLLOWER –> CANDIDATE

Step into the next state and send REQUEST_VOTE message to everybody in the group The number of nodes are saved since the MAJORITY of votes is needed

Note: +1 is added becuase the current node is not included in the GroupMemberCount();

Set the first election Term

The node votes to itself, save the timestamp to filter possibly old votes

Waiting for responses until the MAX election timeout.

If the node is candidate and the wait time (vote time when candidate) has expired It means that nobody responded to the vote request.

If no incoming message for the leader, then the leader just heartbeating Note: The receivers don't respond to AppendEndtry (no log replication in RIAPS so far, no reason to implement the reply)

If no incoming message for the leader and no timeout for sending heartbeat, then maintain the things. like: Checking ongoing vote timeouts
